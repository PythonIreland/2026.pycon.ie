## Project Overview

PyCon Ireland 2026 conference website — a Hugo static site with Tailwind CSS v4 native integration and comprehensive SEO structured data.

**Event:** 17 October 2026 · Trinity College Dublin  
**Deployment:** Netlify (production at `https://2026.pycon.ie/`)


## Architecture

### Hugo + Tailwind v4 Native Integration

This project uses Hugo's **native Tailwind CSS processing** without PostCSS or Node.js build tools:

- `assets/css/main.css` — Tailwind v4 entry point with `@theme` directive for design tokens
- `hugo_stats.json` — Generated by Hugo's `buildStats`, tracks CSS class usage for tree-shaking
- `layouts/partials/css.html` — Uses `templates.Defer` to defer CSS rendering until all classes are collected

**How it works:**
1. Hugo scans all templates and generates `hugo_stats.json` with used classes
2. Tailwind CSS reads this via `@source "hugo_stats.json"` directive
3. Hugo's `css.TailwindCSS` pipe processes the CSS natively
4. CSS is minified and fingerprinted in production builds

### Data-Driven Content Model

Conference content is stored in **YAML data files** (`data/*.yml`), not Markdown frontmatter:

**`data/speakers.yml`**
- Speaker profiles: id, name, title, company, bio, photo, keynote flag, social links
- Accessed: `.Site.Data.speakers`
- Keynotes: `where .Site.Data.speakers "keynote" true`

**`data/sessions.yml`**
- Talk details: id, title, description, speaker_ids (array), type, duration, track
- Types: keynote, talk, break, lightning, ceremony
- Linked to speakers via `speaker_ids`

**`data/schedule.yml`**
- Multi-day, multi-track schedule grid
- Structure: `days[].slots[]` with time, type, session_ids, tracks, duration
- References sessions by ID

**`data/sponsors.yml`**
- Sponsor tiers: Platinum, Gold, Silver, Community
- Each sponsor: name, logo, website, tier

**`data/team.yml`**
- Organising team members

**`data/site.yml`**
- About section content (feature cards)

**`data/dublin.yml`**
- "Discover Dublin" section: intro text, gallery images, highlights (attractions), transport options
- Structure: `intro`, `gallery[]`, `highlights[]`, `transport[]`
- Accessed: `.Site.Data.dublin`

**Why data files?** Allows non-technical team members to update content without touching templates. Enables programmatic manipulation and structured data generation.

### SEO Structured Data System

The site implements **comprehensive JSON-LD schemas** across three layers:

**Layer 1: Homepage (`layouts/partials/structured-data.html`)**
- Event schema: Full conference details (dates, location, pricing, organizer)
- Organization schema: Python Ireland with contact info
- WebSite schema: Site metadata
- BreadcrumbList schema: Navigation hierarchy for interior pages

**Layer 2: Speakers (`layouts/partials/structured-data-speakers.html`)**
- ItemList schema: All speakers with position indexing
- Person schemas: Individual speaker profiles with `performerIn` property
- Bidirectional linking: Speaker → talks, talks → speakers
- Dedicated schemas for keynote speakers with expertise tags

**Layer 3: Schedule (`layouts/partials/structured-data-schedule.html`)**
- EducationalEvent schema: Each talk/keynote with precise timing
- EventSeries schema: Full conference schedule
- `superEvent` hierarchy: Links sessions to main event
- ISO 8601 dates/durations (e.g., `PT30M` for 30 minutes)

**Key Pattern:** Use `@id` for cross-references (e.g., `{{ .Site.BaseURL }}speakers/#{{ $speaker.id }}`). This enables Google Knowledge Graph integration.

**Documentation:** See `docs/STRUCTURED-DATA.md` for complete schema details and validation instructions.

### Configuration (`hugo.toml`)

**Key parameters:**
- `[params]` — Conference details (dates, venue, contact, social). Key social params: `twitterHandle`, `mastodonHandle` (`@Python_Ireland`), `mastodonUrl` (`https://mastodon.ie/@Python_Ireland` — instance is mastodon.ie, not mastodon.social), `youtubeUrl`
- `[params.hero]` — Hero section content
- `[params.stats]` — Statistics for counters (attendees, speakers, tracks, days)
- `[[params.tickets]]` — Ticket types with pricing (Student, Early Bird, General, Corporate)
- `[menus.main]` — Navigation menu items (Sponsors menu currently commented out)

**Build configuration:**
- Cache busters for CSS changes (tailwind/postcss config, hugo_stats.json)
- `buildStats.enable = true` — Required for Tailwind class tracking

## Content Management

### Adding Speakers

Edit `data/speakers.yml`:
```yaml
- id: "speaker-slug"
  name: "Full Name"
  title: "Job Title"
  company: "Company Name"
  bio: "Biography text"
  photo: "/img/speakers/photo.jpg"  # or placeholder.svg
  keynote: true  # or false
  social:
    twitter: "https://twitter.com/handle"
    github: "https://github.com/username"
    linkedin: "https://linkedin.com/in/username"
```

Structured data will auto-generate for new speakers.

### Adding Sessions/Talks

Edit `data/sessions.yml`:
```yaml
- id: "talk-slug"
  title: "Talk Title"
  description: "Detailed description"
  speaker_ids: ["speaker-slug"]  # Array, can have multiple
  type: "talk"  # or "keynote", "lightning", "workshop"
  duration: 30  # minutes
  track: "track-1"  # or "main" for plenary
```

Then add to schedule in `data/schedule.yml`:
```yaml
- time: "14:00"
  type: "session"
  session_ids: ["talk-slug"]
  tracks: ["track-1"]
  duration: 30
```

EducationalEvent schemas will auto-generate with correct dates/times.

### Adding Blog Posts

Create `content/blog/post-slug.md`:
```markdown
---
title: "Post Title"
date: 2026-03-01
description: "Meta description for SEO"
---

Markdown content here.
```

Posts appear on `/blog/` automatically.

## Common Patterns

### Accessing Data Files in Templates

```go
{{ range .Site.Data.speakers }}
  {{ .name }} — {{ .company }}
{{ end }}

{{ $keynotes := where .Site.Data.speakers "keynote" true }}

{{ range where .Site.Data.sessions "type" "talk" }}
  {{ .title }}
{{ end }}
```

### Linking Sessions to Speakers

```go
{{ range .speaker_ids }}
  {{ $speakerId := . }}
  {{ range where $.Site.Data.speakers "id" $speakerId }}
    <p>{{ .name }}</p>
  {{ end }}
{{ end }}
```

### Conditional Rendering

```go
{{ if eq .type "keynote" }}
  <span class="badge">Keynote</span>
{{ end }}

{{ if .Site.Params.tickets }}
  {{ range .Site.Params.tickets }}
    {{ if .highlight }}
      <div class="ticket-highlight">...</div>
    {{ end }}
  {{ end }}
{{ end }}
```
